Naming & structure

  • Use simple, predictable naming patterns so sites are easy to identify (e.g., HR-, FIN-, PRJ-).
  • Group related sites under hubs so information is connected instead of scattered.
  • Avoid creating duplicate sites; check if one already exists before starting a new workspace.
  • Keep navigation intuitive by avoiding deep folder structures and unused pages.

For examples and official conventions, visit GNB Technology Services | SharePoint .
